# Provenance Guard — Portfolio Walkthrough Script (~4–5 min)

Format: **[SAY]** = say to camera · **[DO]** = do on screen. Keep it relaxed and unpolished.
Have two things open before you hit record: (1) a terminal with the server running, (2) your editor
showing `planning.md` and `app.py`.

**Setup before recording (off-camera):**
```bash
cd "Project 4"
source .venv/bin/activate
python app.py            # leave running on http://localhost:5000
```
Open a SECOND terminal tab for the curl commands below.

---

## 0. Intro — 20 sec
**[SAY]** "Hi, this is my walkthrough of **Provenance Guard** — a backend service a writing or art
platform could plug in to estimate whether a piece of text was written by a human or generated by
AI. The whole thing is built around one principle: on a creative platform, **wrongly accusing a
real author of using AI is the worst possible outcome** — so the system is designed to be slow to
accuse and quick to say 'I'm not sure.'"

---

## 1. Architecture — 40 sec
**[DO]** Show the `## Architecture` diagram in `planning.md`.
**[SAY]** "A submission comes in to `POST /submit`. It passes a rate limiter, then runs through
**three independent detection signals**: a **semantic** signal — a Groq LLM reading the voice and
register; a **structural** signal — pure-Python stylometry measuring sentence-length variance and
punctuation; and a **lexical** signal — surface AI-cliché versus casual-human phrasing. Those get
fused into two numbers — *which way* the evidence leans, and *how confident* we are — which map to a
plain-language label. Every decision is written to a structured audit log. And there's a second
flow, `POST /appeal`, for creators to contest a verdict."

---

## 2. Demo: clearly-AI text — 50 sec
**[DO]** Run:
```bash
curl -s -X POST http://localhost:5000/submit \
  -H "Content-Type: application/json" \
  -d '{"text": "In todays rapidly evolving digital landscape, organizations must leverage innovative solutions to remain competitive. Furthermore, it is important to note that stakeholders across various sectors must collaborate to ensure optimal outcomes. Moreover, robust strategic initiatives play a crucial role in driving sustainable growth.", "creator_id": "demo-ai"}' | python -m json.tool
```
**[SAY]** "Here's obviously-AI text. Notice the response: `attribution` is **likely_ai**, a high
`confidence`, and — the key part — I get back **every individual signal score**, not just one
number, so the verdict is transparent. And the `label` is written in plain English a normal reader
could understand — no jargon."

## 3. Demo: clearly-human text — 30 sec
**[DO]** Run:
```bash
curl -s -X POST http://localhost:5000/submit \
  -H "Content-Type: application/json" \
  -d '{"text": "ok so i finally tried that new ramen place downtown and honestly? underwhelming. the broth was fine but they put WAY too much sodium in it and i was thirsty for like three hours after. my friend got the spicy version and said it was better. probably won'\''t go back unless someone drags me there", "creator_id": "demo-human"}' | python -m json.tool
```
**[SAY]** "Casual human writing — now it flips to **likely_human** with a completely different label.
So confidence and the label genuinely vary with the input; it's not a constant."

---

## 4. The design decision that matters: false positives — 45 sec
**[DO]** Run (and copy the `content_id` from the output — you'll need it in step 5):
```bash
curl -s -X POST http://localhost:5000/submit \
  -H "Content-Type: application/json" \
  -d '{"text": "The relationship between monetary policy and asset price inflation has been extensively studied in the literature. Central banks face a fundamental tension between their mandate for price stability and the unintended consequences of prolonged low interest rates on equity and real estate valuations.", "creator_id": "demo-formal"}' | python -m json.tool
```
**[SAY]** "This is the interesting case — **formal human writing**, the kind a non-native English
speaker or an academic might produce. Look: `combined_p_ai` is actually **above 0.5 — it leans
AI**. But the label says **'Attribution uncertain,' not an accusation.** That's deliberate. The
signals partly disagree, which drops confidence below my 0.70 gate, and a low-confidence lean
collapses to 'uncertain.' That's the false-positive protection working end to end — I'd rather flag
uncertainty than wrongly accuse a real writer."

---

## 5. Appeals — 40 sec
**[DO]** Paste the `content_id` from step 4 into:
```bash
curl -s -X POST http://localhost:5000/appeal \
  -H "Content-Type: application/json" \
  -d '{"content_id": "PASTE-CONTENT-ID-HERE", "creator_reasoning": "I wrote this myself. I am a non-native English speaker and my register is formal, which the detector misreads as AI."}' | python -m json.tool
```
**[SAY]** "If a creator still disagrees, they appeal with their reasoning. The system flips that
content's status to **under_review** and logs the appeal right next to the original decision for a
human reviewer."

---

## 6. Rate limiting — 20 sec
**[DO]** Run:
```bash
for i in $(seq 1 12); do curl -s -o /dev/null -w "%{http_code} " -X POST http://localhost:5000/submit -H "Content-Type: application/json" -d '{"text":"rate limit test sentence.","creator_id":"rl"}'; done; echo
```
**[SAY]** "Rate limiting: ten per minute, a hundred per day. A real creator posting their own work
never hits that, but a script flooding the endpoint does — you can see the first ten succeed with
200s, then it starts returning 429s."

---

## 7. Audit log + analytics — 35 sec
**[DO]** Run:
```bash
curl -s "http://localhost:5000/log?limit=5" | python -m json.tool
curl -s http://localhost:5000/analytics | python -m json.tool
```
**[SAY]** "Everything's captured in a structured audit log — timestamp, attribution, confidence, all
three signal scores, and the appeal sitting alongside the original entry. And as a stretch feature,
`/analytics` rolls the log up into platform metrics — the verdict mix, the appeal rate, and average
confidence."

---

## 8. Wrap — 25 sec
**[SAY]** "So to recap the design decisions: three independent signals instead of one, two numbers
that separate 'which way' from 'how sure,' a confidence gate that protects human writers, and an
appeal path for when we get it wrong. If I were deploying this for real, the next steps would be
real user auth on appeals and moving the log to a database. Thanks for watching."
